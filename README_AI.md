# AI Chat Assistant

This repository includes an optional collapsible AI chat window that lets
an assistant work inside the Fantasy Map Generator as a regular user — using
the same UI affordances you would. The assistant is powered by the
[Anthropic Claude API](https://docs.anthropic.com/).

## What it can do today

Each iteration of development adds one new capability ("use case"). The
currently implemented tools are:

| Tool                    | What the AI can do                                                            | Example prompt                            |
| ----------------------- | ----------------------------------------------------------------------------- | ----------------------------------------- |
| `set_map_name`          | Rename the current map (updates `#mapName` field)                             | "Rename my map to Eldoria"                |
| `set_label_text`        | Rewrite the text of a specific on-map label — state label (`stateLabel{i}`), burg label (`burgLabel{i}`), or custom/added label (`label{i}`). Same side-effect as typing into the Labels Editor's text field: splits input on `\|` for multi-line, rebuilds the `<textPath>`'s inner tspans (single-line: one `<tspan x="0">`; multi-line: `<tspan x="0" dy="{offset}em">` per segment, mirroring `labels-editor.js:changeText`). Matches the target by DOM id first; falls back to exact current pipe-joined text (ambiguous matches error with candidate ids so you can disambiguate). DOM-only — there is no `pack.labels` array; does NOT rename the underlying state / burg data (use `rename_state` / `rename_burg` for that). | "Change the Fantasy Map label to 'Eldoria'", "Rename stateLabel3 to 'Ashmark \| Empire'" |
| `set_layer_visibility`  | Show or hide a named map layer (rivers, borders, states, religions, cultures, heightmap, biomes, ice, labels, burgs, routes, markers, relief, emblems, zones, grid, coordinates, compass, temperature, precipitation, population, scale bar, vignette, texture, rulers). Idempotent. | "Turn off the rivers", "Show religions", "Hide state borders" |
| `apply_layers_preset`   | Apply a named preset: political, cultural, religions, provinces, biomes, heightmap, physical, poi, military, emblems, landmass. Accepts aliases like "culture map", "religion", "minimalist". | "Switch to the political map", "Show me the cultural view" |
| `set_layer_opacity`     | Set the opacity of a named SVG map layer — mirrors the Style Editor's opacity slider (`styleOpacityInput`). Writes the `opacity` attribute on the layer's `<g>` element (e.g. `#biomes`, `#rivers`, `#markers`, `#terrs` for heightmap, `#regions` for states, `#cults` for cultures, `#relig` for religions, `#burgIcons` for burgs). `opacity` is a float in [0, 1] (matches the UI slider's `min=0 max=1 step=0.01`); 1 = fully opaque, 0 = fully transparent. Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_visibility` (hide/show) and `apply_layers_preset` (bulk on/off). Returns `{previousOpacity, opacity}`. Requires an Anthropic API key (see "Getting an API key" below). | "Set the biomes layer opacity to 0.5", "Make the markers semi-transparent — opacity 0.6", "Dim the heightmap to 0.3" |
| `set_layer_stroke_width`| Set the `stroke-width` of a named SVG map layer — mirrors the Style Editor's stroke-width slider (`styleStrokeWidthInput`). Writes the `stroke-width` attribute on the layer's `<g>` element; SVG children inherit the value unless they override it, so this works for both direct-stroke layers (borders, rivers, routes) and containers (heightmap, states). `width` is a float in [0, 10] (matches the UI slider's `min=0 max=10 step=0.01`). Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_opacity` (alpha), `set_layer_visibility` (hide/show), and `apply_layers_preset` (bulk on/off). Returns `{previousWidth, width}` (`previousWidth` is `null` when the attribute was absent or unparseable). Requires an Anthropic API key (see "Getting an API key" below). | "Make the borders thicker — stroke width 1.5", "Set the rivers stroke width to 0.5", "Thin the routes to 0.3" |
| `set_layer_stroke_color`| Set the `stroke` (color) of a named SVG map layer — mirrors the Style Editor's stroke color picker (`styleStrokeInput`). Writes the `stroke` attribute on the layer's `<g>` element; SVG children inherit the value unless they override it, so this works for both direct-stroke layers (borders, rivers, routes) and containers (heightmap, states). `stroke` is any valid CSS color: `#rgb` / `#rrggbb` / `#rrggbbaa` hex, `rgb()` / `rgba()` / `hsl()` / `hsla()` / `hwb()` / `lab()` / `lch()` / `color()` functions, or a named color (e.g. `navy`, `tomato`). Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_stroke_width` (thickness), `set_layer_opacity` (alpha), `set_layer_visibility` (hide/show), and `apply_layers_preset` (bulk on/off). Returns `{previousStroke, stroke}` (`previousStroke` is the raw prior attribute value or `null` when absent). Requires an Anthropic API key (see "Getting an API key" below). | "Make the river strokes blue", "Set the state borders stroke to #330000", "Change the routes color to rgba(120, 40, 10, 0.8)" |
| `set_layer_stroke_dasharray`| Set the `stroke-dasharray` of a named SVG map layer — mirrors the Style Editor's stroke-dasharray input (`styleStrokeDasharrayInput` → `getEl().attr("stroke-dasharray", …)` in `public/modules/ui/style.js`). Writes the `stroke-dasharray` attribute on the layer's `<g>` element; SVG children inherit the value unless they override it, so this works for both direct-stroke layers (borders, rivers, routes) and containers (heightmap, states). `dasharray` is an SVG stroke-dasharray string — space- or comma-separated non-negative finite numbers (e.g. `"5,5"`, `"2 4 2"`, `"10 5"`, `"1,2,3,4"`); pass an empty string or `"none"` to clear the dash pattern and return to solid strokes. Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_stroke_width` (thickness), `set_layer_stroke_color` (color), `set_layer_opacity` (alpha), `set_layer_visibility` (hide/show), and `apply_layers_preset` (bulk on/off). Returns `{previousDasharray, dasharray}` (`previousDasharray` is the raw prior attribute value or `null` when absent; `dasharray` is `""` when cleared, otherwise the trimmed input). Requires an Anthropic API key (see "Getting an API key" below). | "Make the state borders dashed — 5,5", "Set the routes stroke dasharray to 2 4 2", "Clear the dashed pattern on the rivers" |
| `set_font_family`       | Set the `font-family` of a text-bearing SVG layer group — mirrors the Style Editor's per-layer font picker (`styleSelectFont.on("change", …)`). Writes the `font-family` attribute on the layer's `<g>`; child `<text>` / `<textPath>` inherit it (no redraw needed). Supported `layer` values: `labels` (#labels), `state_labels` (#states), `added_labels` (#addedLabels — custom user labels), `burg_labels` (#burgLabels), `province_labels` (#provs), `legend` (#legend), or `all` which applies across #labels, #provs, and #legend (the three groups `getUsedFonts` in `src/modules/fonts.ts` collects from). `font` is any CSS font-family string (e.g. `"Almendra SC"`, `"Garamond"`, `"Forum"`) — the font must already be loaded via the Style Editor's custom-font dialog or a default preset. Returns `{previousFont, font}`, plus an `applied` array of per-layer previous values when `layer="all"`. Requires an Anthropic API key (see "Getting an API key" below). | "Change the state label font to Almendra SC", "Use Garamond for all map labels", "Set the burg labels font to Forum" |
| `set_font_size`         | Set the `font-size` of a text-bearing SVG layer group — mirrors the Style Editor's per-layer font-size slider (`styleFontSize.on("change", …)` → `changeFontSize`). Writes both `data-size` (the user-authored size preserved across zoom-driven rescaling) and `font-size` on the layer's `<g>`; child `<text>` / `<textPath>` inherit `font-size`. Supported `layer` values mirror `set_font_family`: `labels` (#labels), `state_labels` (#states), `added_labels` (#addedLabels), `burg_labels` (#burgLabels), `province_labels` (#provs), `legend` (#legend), or `all` which applies across #labels, #provs, and #legend. `size` is a number in [0.5, 100] (matches the UI input's `min=.5 max=100 step=.1`); typical map label sizes are 6–20. Returns `{previousSize, size}`, plus an `applied` array of per-layer previous values when `layer="all"`. Requires an Anthropic API key (see "Getting an API key" below). | "Make the state labels bigger — font size 24", "Shrink the burg labels to 8", "Set the legend font size to 14" |
| `set_layer_filter`      | Apply an SVG filter to a named map layer — mirrors the Style Editor's per-layer filter dropdown (`styleFilterInput.on("change", …)` in `public/modules/ui/style.js`). Writes `filter="url(#<id>)"` on the layer's `<g>` element, or `filter=""` to clear. Valid ids are the 20 filters defined in `<g id="filters">` inside `src/index.html`: `blurFilter`, `blur1`, `blur3`, `blur5`, `blur7`, `blur10`, `splotch`, `bluredSplotch`, `dropShadow`, `dropShadow01`, `dropShadow05`, `outline`, `pencil`, `turbulence`, `paper`, `crumpled`, `filter-grayscale`, `filter-sepia`, `filter-dingy`, `filter-tint`. Friendly aliases: `sepia` → `filter-sepia`, `grayscale` / `greyscale` → `filter-grayscale`, `dingy` → `filter-dingy`, `tint` → `filter-tint`, `shadow` / `drop shadow` → `dropShadow`, `blur` → `blur3`. Pass `filter=""` or `filter="none"` to clear. Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_opacity` (alpha), `set_layer_stroke_width`, `set_layer_visibility` (hide/show), and `apply_layers_preset`. Returns `{previousFilter, filter}` (`previousFilter` is the raw attribute string or `null`). Requires an Anthropic API key (see "Getting an API key" below). | "Add a drop shadow to the rivers", "Apply a sepia filter to the biomes", "Clear the filter on the markers" |
| `set_layer_fill`        | Set the `fill` color of a named SVG map layer — mirrors the Style Editor's per-layer fill color picker (`styleFillInput.on("input", …)` in `public/modules/ui/style.js`). Writes the `fill` attribute on the layer's `<g>` element (e.g. `#biomes`, `#rivers`, `#markers`, `#terrs` for heightmap, `#regions` for states); SVG children inherit `fill` unless they override it. Accepts hex (`#rgb`, `#rgba`, `#rrggbb`, `#rrggbbaa`), functional colors (`rgb()`, `rgba()`, `hsl()`, `hsla()`), and common CSS named colors (e.g. `red`, `cornflowerblue`). Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Distinct from `set_layer_opacity` (alpha), `set_layer_stroke_width`, `set_layer_filter`, and `set_layer_visibility` (hide/show). Returns `{previousFill, fill}` (`previousFill` is the raw attribute string or `null` when absent). Requires an Anthropic API key (see "Getting an API key" below). | "Make the rivers bright blue", "Set the biomes fill to #336699", "Paint the markers red" |
| `get_layer_style`       | Read back the current style attributes of a named SVG map layer — the read-side companion to the `set_layer_*` style family (`set_layer_opacity`, `set_layer_fill`, `set_layer_stroke_color`, `set_layer_stroke_width`, `set_layer_stroke_dasharray`, `set_layer_filter`). Reads the `opacity`, `fill`, `stroke`, `stroke-width`, `stroke-dasharray`, and `filter` attributes directly from the layer's `<g>` element (e.g. `#biomes`, `#rivers`, `#markers`, `#terrs` for heightmap, `#regions` for states, `#cults` for cultures, `#relig` for religions, `#burgIcons` for burgs). Numeric attrs (`opacity`, `stroke-width`) are parsed to numbers — unparseable or absent values return `null`. String attrs (`fill`, `stroke`, `stroke-dasharray`, `filter`) return the raw attribute string, or `null` when absent / empty. Layer names / aliases mirror `set_layer_visibility` (rivers, biomes, markers, state borders, heightmap, burgs, cultures, religions, …). Read-only — never mutates the DOM. Returns `{layer, opacity, fill, stroke, strokeWidth, strokeDasharray, filter}`. Errors when the layer name is unknown or the element is not present in the DOM. Requires an Anthropic API key (see "Getting an API key" below). | "What's the current opacity of the biomes layer?", "Show me the stroke style on the state borders", "Does the rivers layer have a filter applied?" |
| `set_style_preset`      | Switch the map's visual theme — same as the Options panel's Style Preset selector. One of: `default`, `ancient`, `gloom`, `pale`, `light`, `watercolor`, `clean`, `atlas`, `darkSeas`, `cyberpunk`, `night`, `monochrome` (case-insensitive). Delegates to `window.changeStyle(preset)` which applies the preset's style JSON across all SVG elements and re-renders burg icons / labels. Custom user-saved presets (`fmgStyle_*`) are out of scope. | "Switch to the ancient theme", "Use the monochrome style" |
| `list_style_presets`    | List every style preset identifier accepted by `set_style_preset` — the discovery companion to that tool. Returns the 12 built-in themes (`default`, `ancient`, `gloom`, `pale`, `light`, `watercolor`, `clean`, `atlas`, `darkSeas`, `cyberpunk`, `night`, `monochrome`) plus any user-saved custom presets stored in `localStorage` under the `fmgStyle_` prefix (same set the Options panel's Style Preset `<select>` renders via `public/modules/ui/style-presets.js`). Each entry is `{id, name, builtin}` — pass `id` verbatim to `set_style_preset` (built-in ids are case-insensitive; custom ids are exact). Built-ins come first in canonical order; customs follow sorted by id ascending. Also returns `count` = total entries. Takes no parameters. Read-only. Requires an Anthropic API key (see "Getting an API key" below). | "List the style presets", "Which themes can I switch to?", "Show me the custom saved styles" |
| `get_map_info`          | Read a summary of the current map: name, seed, mapId, dimensions, year/era, and counts of states, provinces, burgs, religions, cultures, rivers, markers, zones, cells. | "What's this map called?", "How many states are there?", "Summarize the world" |
| `get_population_stats`  | Read aggregate population statistics for the whole generated world — the demographic summary parallel of `get_map_info`. Sums rural population as `pack.cells.pop[k] × populationRate` over every cell (typed-array scan) and urban population as `burg.population × populationRate × urbanization` over every non-removed burg (skipping the index-0 placeholder), matching the scaling `list_burgs` / `list_states` / `get_state_info` / `get_province_info` use for display. Also surfaces the top N most populous states (from pre-aggregated `state.rural + state.urban`, skipping id 0 Neutrals and removed states) and top N most populous burgs (sorted desc by computed population, skipping id 0 and removed). Optional `top_n` (integer in [0, 500], default 10) — `0` returns empty top lists. Returns `{ ok, population_total, urban_population, rural_population, top_states, top_burgs, population_rate, urbanization }` with integer-rounded populations (each top-list entry is `{ i, name, population }`). Read-only — never mutates the pack. Useful for demographic audits, picking expansion targets, or sanity-checking `set_world_rates` changes. Errors only on un-generated map or out-of-range `top_n`. Requires an Anthropic API key (see "Getting an API key" below). | "How many people live on this map?", "Top 5 biggest cities", "What's the urban vs. rural split?", "Which states are the most populous?" |
| `get_religion_distribution` | Read the aggregate distribution of religions across the current map — the religion-level parallel of `get_population_stats` and a companion to `list_religions` / `find_largest_religions`. Iterates `pack.religions` linearly, skipping the index-0 "No religion" placeholder and any `removed: true` entries, and uses the pre-aggregated per-religion fields (`religion.cells`, `religion.area`, `religion.rural + religion.urban`) — no per-cell scan, so it stays O(religions). Followers are computed as `Math.round((rural + urban) × populationRate)` with the same `rate <= 0 / NaN → 1` fallback `list_religions` uses (but NOT multiplied by `urbanization`, since a religion's `rural + urban` is already the combined raw total). `percentage` is each religion's share of `total_cells` — `cells_count / total_cells × 100`, floating, `0` when `total_cells` is `0`. Returns `{ ok, total_cells, total_followers, religions }` sorted by `cells_count` descending. Each religion is `{ i, name, color, type, form, cells_count, percentage, area, followers }`; `color` / `type` / `form` fall back to `null` when the raw religion omits them. `total_cells` is the sum of `religion.cells` over active religions (NOT the total map cell count — cells without an organized religion sit on id 0 and are excluded). `total_followers` is the sum of the per-religion scaled follower counts. Takes no parameters. When the map has no active religions, `religions` is `[]`, `total_cells` and `total_followers` are `0`, still `ok: true`. Useful for audits (how is faith distributed across the map?), demographic summaries that need per-religion follower counts, and deciding which religions to rename / recolor / retype. Read-only — never mutates the pack. Errors only on un-generated map. Requires an Anthropic API key (see "Getting an API key" below). | "How are religions distributed on this map?", "What percentage of cells follow the Ashenfaith?", "Which religion has the most followers?", "Give me a religion-by-religion breakdown" |
| `get_culture_distribution` | Read the aggregate distribution of cultures across the current map — the culture-level parallel of `get_population_stats` / `get_religion_distribution` / `get_biome_distribution` and a companion to `list_cultures` / `find_largest_cultures`. Iterates `pack.cultures` linearly, skipping only `removed: true` entries and **INCLUDING culture id 0 (Wildlands)** — unlike religions / states / burgs where id 0 is a skipped placeholder, the Wildlands slot is a real culture that the Culture Editor and `get_culture_info` treat as readable, so it shows up in the distribution too. Uses the pre-aggregated per-culture fields (`culture.cells`, `culture.area`, `culture.rural + culture.urban`) — no per-cell scan, so it stays O(cultures). Population is computed as `Math.round((rural + urban) × populationRate)` with the same `rate <= 0 / NaN → 1` fallback `list_cultures` uses (NOT multiplied by `urbanization`, since a culture's `rural + urban` is already the combined raw total — same convention `get_religion_distribution` follows). `percentage` is each culture's share of `total_cells` — `cells_count / total_cells × 100`, floating, `0` when `total_cells` is `0`. Returns `{ ok, total_cells, total_population, cultures }` sorted by `cells_count` descending. Each culture is `{ i, name, color, type, cells_count, percentage, area, population }`; `color` / `type` fall back to `null` when the raw culture omits them. `total_cells` is the sum of `culture.cells` over non-removed cultures (INCLUDING Wildlands — cells belonging to the Wildlands slot are still counted here, differently from the religion distribution which excludes id 0). `total_population` is the sum of the per-culture scaled population counts. Takes no parameters. When the map has no non-removed cultures, `cultures` is `[]`, `total_cells` and `total_population` are `0`, still `ok: true`. Useful for audits (how is cultural territory distributed across the map?), demographic summaries that need per-culture population counts, and deciding which cultures to rename / recolor / retype. Read-only — never mutates the pack. Errors only on un-generated map. Requires an Anthropic API key (see "Getting an API key" below). | "How are cultures distributed on this map?", "What percentage of cells are Highlanders?", "Which culture has the most people?", "Give me a culture-by-culture breakdown" |
| `get_state_distribution` | Read the aggregate distribution of states across the current map — the state-level parallel of `get_religion_distribution` / `get_culture_distribution` and a companion to `list_states` / `find_largest_states`. Iterates `pack.states` linearly, skipping the index-0 "Neutrals" placeholder and any `removed: true` entries, and uses the pre-aggregated per-state fields (`state.cells`, `state.area`, `state.rural + state.urban`) — no per-cell scan, so it stays O(states). Population is computed as `Math.round((rural + urban) × populationRate)` with the same `rate <= 0 / NaN → 1` fallback `list_states` uses (but NOT multiplied by `urbanization`, since a state's `rural + urban` is already the combined raw total). Capital name is resolved via `pack.burgs[state.capital].name` (falls back to `null` when `state.capital` is `0` or the burg is missing) — matches `list_states` / `find_largest_states`. `percentage` is each state's share of `total_cells` — `cells_count / total_cells × 100`, floating, `0` when `total_cells` is `0`. Returns `{ ok, total_cells, total_population, states }` sorted by `cells_count` descending. Each state is `{ i, name, fullName, form, color, capital, cells_count, percentage, area, population }`; `fullName` / `form` / `color` / `capital` fall back to `null` when the raw state omits them. `total_cells` is the sum of `state.cells` over active states (NOT the total map cell count — cells without an assigned state sit on Neutrals id 0 and are excluded). `total_population` is the sum of the per-state scaled populations. Takes no parameters. When the map has no active states, `states` is `[]`, `total_cells` and `total_population` are `0`, still `ok: true`. Useful for audits (how is territory distributed across polities?), demographic summaries that need per-state population counts, and deciding which states to rename / recolor / merge. Read-only — never mutates the pack. Errors only on un-generated map. Requires an Anthropic API key (see "Getting an API key" below). | "How are states distributed on this map?", "What percentage of the land belongs to Altaria?", "Which state controls the most territory?", "Give me a state-by-state breakdown with capitals" |
| `get_state_info`        | Read detailed info for a single state — the per-state parallel of `get_map_info` / `get_cell_info`. Required `state` identifies the state by numeric id (> 0) or case-insensitive name / fullName (resolved via `findEntityByRef`; skips the Neutrals placeholder at id 0 and any `removed: true` entries). Returns `i`, `name`, `fullName`, `form`, `formName`, `type`, `color`, `expansionism`, resolved `culture` ({id, name} from `pack.cultures`), `capital` ({id, name, x, y} from `pack.burgs` — `null` when `state.capital` is 0 / missing), `center` ({cell, x, y} from `pack.cells.p` — `null` when `state.center` is absent), `cells_count` (`state.cells`), `area` (`state.area`), `population_total` / `urban_population` / `rural_population` (rounded, using `populationRate × urbanization` the same way `list_states` / `list_burgs` scale), `burgs_count` (live count of non-removed burgs where `burg.state === i`), a `provinces` list ({id, name}) of non-removed provinces assigned to this state, the raw `diplomacy` row (string[] indexed by the other state's id — cross-walk with `list_states` / `list_diplomacy` for the other state's name), and `lock`. Useful before taking any state-targeted action (`rename_state`, `set_state_capital`, `set_state_color`, `set_diplomacy`, `merge_states`, `regenerate_state_name`, `regenerate_state_coa`, …). Errors on un-generated map, state 0 (Neutrals), or an unresolvable ref. Requires an Anthropic API key (see "Getting an API key" below). | "Describe Altaria in detail", "What's the capital and population of state 3?", "Give me the full info on the Kingdom of Valorin — provinces, culture, diplomacy" |
| `get_religion_info`     | Read detailed info for a single religion — the per-religion parallel of `get_state_info` / `get_culture_info`. Required `religion` identifies the religion by numeric id (> 0) or case-insensitive name (resolved via `findEntityByRef`; skips the "No religion" placeholder at id 0 and any `removed: true` entries). Returns `i`, `name`, `color`, `type` (Folk / Organized / Cult / Heresy / …), `form`, `deity`, `code`, `expansion` (global / state / culture mode), `expansionism`, resolved `culture` ({id, name} from `pack.cultures`, `null` when `religion.culture` is unset), `center` ({cell, x, y} from `pack.cells.p` — `null` when `religion.center` is absent), `origins` (raw parent-religion id array from `religion.origins`), `cells_count` (`religion.cells`), `area` (`religion.area`), `population_total` / `urban_population` / `rural_population` (rounded, using `populationRate × urbanization` the same way `list_religions` / `list_burgs` scale), `burgs_count` (live count of non-removed burgs whose cell's `religion` slot matches — religions aren't stored on burgs directly), `states_count` (best-effort count of non-removed states whose capital-burg sits on a cell carrying this religion), and `lock`. Useful before taking any religion-targeted action (`rename_religion`, `set_religion_color`, `set_religion_type`, `set_religion_form`, `set_religion_deity`, `set_religion_expansion`, `set_religion_culture`, `set_religion_center`, `set_religion_origins`, `regenerate_religion_names`, `remove_religion`, …). Errors on un-generated map, religion 0 ("No religion"), or an unresolvable ref. Requires an Anthropic API key (see "Getting an API key" below). | "Describe the Ashenfaith religion in detail", "What's the center and population of religion 3?", "Give me the full info on the Cult of the Silver Moon — deity, origins, expansion" |
| `get_cell_info`         | Read every meaningful property of a single packed-grid cell — the per-cell parallel of `get_map_info`. Required `cell` (integer `[0, pack.cells.i.length)`). Returns `height` (0-100), `biome` ({id, name via `biomesData.name`}), `temperature` (°C) and `precipitation` resolved via `pack.cells.g[cell]` → `grid.cells.temp`/`prec` (same indirection the `cellInfo` tooltip uses in `public/modules/ui/general.js:updateCellInfo`), `population` (`cells.pop[cell]`), `x`/`y` from `cells.p[cell]`, resolved `{id,name}` refs for `state` / `province` / `culture` / `religion` (neutral slot returns `{id:0,name:…}`), `burg` / `river` (returns `null` when the cell's `burg` / `r` slot is 0; rivers are looked up by `.i`), `feature` ({id, type, group, land, border, name} for the landmass / ocean / lake — `null` when `cells.f[cell]` is 0), and `neighbors` (plain-array copy of `cells.c[cell]`). Useful before taking any cell-targeted action (`add_burg`, `add_culture`, `add_marker`, `add_regiment`, …). Errors on out-of-range `cell`, non-integer `cell`, or an un-generated map. ## Getting an API key

1. Sign in at <https://console.anthropic.com/>.
2. Go to **API Keys** and click **Create Key**.
3. Copy the generated key (it starts with `sk-ant-…`).

You will be billed for your own usage on your own Anthropic account.

## Using the chat in the app

1. Start the app (`npm run dev`, then open <http://localhost:5173>) or use a
   built version.
2. In the bottom-right corner, click the small circular **AI** button.
3. The first time the panel opens, an API-key row is expanded. Paste your
   `sk-ant-…` key and click **Save**. The key is stored in your browser's
   `localStorage` under the name `ai-chat-anthropic-api-key` — nothing is
   sent anywhere except directly to Anthropic.
4. Type a request in the input box (Shift+Enter for a newline, Enter to
   send). Example: `rename the map to Eldoria`.
5. You'll see:
   - your message (blue bubble),
   - any tool calls the AI makes (`→ set_map_name({"name":"Eldoria"})`,
     `✓ set_map_name → {"ok":true,...}`),
   - the assistant's reply.
6. Click the `–` in the header to collapse the panel back to the AI button.
7. Click the **⚙** in the header to show/hide the API-key row (to change or
   clear the stored key).

## Specifying the API key — other options

The built-in flow uses `localStorage`. For advanced usage:

- **Programmatic override**: call `setApiKey(key)` from the `src/ai/api-key`
  module (exported from `src/ai`).
- **Clearing the key**: call `clearApiKey()` from the same module, or open
  DevTools and run
  `localStorage.removeItem("ai-chat-anthropic-api-key")`.

There is intentionally no support for bundling an API key at build time —
that would ship the key to every visitor of the site.

## Security & limitations

- The API key is visible to any script running in the same page. Do not use
  the feature on a shared computer or on a page where you don't trust the
  other scripts.
- The key is sent directly from your browser to
  `https://api.anthropic.com/v1/messages`. The app has no backend
  intermediary.
- Anthropic requires the header
  `anthropic-dangerous-direct-browser-access: true` for browser calls. We
  set it automatically — it is a deliberate acknowledgement that direct
  browser calls are not the recommended deployment pattern.
- Each message consumes tokens on your Anthropic account. Keep conversations
  focused.

## Troubleshooting

- **"No API key set. Click ⚙…"** — you haven't saved a key yet.
- **401 / authentication_error** — the key is wrong, revoked, or your
  account has no credits.
- **429 / rate limit** — too many requests; slow down or upgrade your plan.
- **CORS errors in DevTools** — make sure you are not proxying the request
  through another host; the call must go directly to
  `api.anthropic.com`.
- **Tool call says `#mapName not found`** — the map input element is only
  present after the main map loads; reload the page and wait for the map to
  finish generating before issuing rename commands.

## Developing new tools

Use cases are implemented one per Ralph-loop iteration. See
`plan_N.md` and `tasks_N.md` at the root of the repo for the active plan.
To add a new tool:

1. Create a file in `src/ai/tools/your-tool.ts` that exports a `Tool`
   (matching the `Tool` interface in `src/ai/tools/index.ts`).
2. Register it from `src/ai/index.ts` inside `buildDefaultRegistry()`.
3. Add a unit test under `src/ai/tools/your-tool.test.ts`.
4. Update this README's tool table with the new capability.
