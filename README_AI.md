# AI Chat Assistant

This repository includes an optional collapsible AI chat window that lets
an assistant work inside the Fantasy Map Generator as a regular user — using
the same UI affordances you would. The assistant is powered by the
[Anthropic Claude API](https://docs.anthropic.com/).

## What it can do today

Each iteration of development adds one new capability ("use case"). The
currently implemented tools are:

| Tool                    | What the AI can do                                                            | Example prompt                            |
| ----------------------- | ----------------------------------------------------------------------------- | ----------------------------------------- |
| `set_map_name`          | Rename the current map (updates `#mapName` field)                             | "Rename my map to Eldoria"                |
| `set_label_text`        | Rewrite the text of a specific on-map label — state label (`stateLabel{i}`), burg label (`burgLabel{i}`), or custom/added label (`label{i}`). Same side-effect as typing into the Labels Editor's text field: splits input on `\|` for multi-line, rebuilds the `<textPath>`'s inner tspans (single-line: one `<tspan x="0">`; multi-line: `<tspan x="0" dy="{offset}em">` per segment, mirroring `labels-editor.js:changeText`). Matches the target by DOM id first; falls back to exact current pipe-joined text (ambiguous matches error with candidate ids so you can disambiguate). DOM-only — there is no `pack.labels` array; does NOT rename the underlying state / burg data (use `rename_state` / `rename_burg` for that). | "Change the Fantasy Map label to 'Eldoria'", "Rename stateLabel3 to 'Ashmark \| Empire'" |
| `set_layer_visibility`  | Show or hide a named map layer (rivers, borders, states, religions, cultures, heightmap, biomes, ice, labels, burgs, routes, markers, relief, emblems, zones, grid, coordinates, compass, temperature, precipitation, population, scale bar, vignette, texture, rulers). Idempotent. | "Turn off the rivers", "Show religions", "Hide state borders" |
| `apply_layers_preset`   | Apply a named preset: political, cultural, religions, provinces, biomes, heightmap, physical, poi, military, emblems, landmass. Accepts aliases like "culture map", "religion", "minimalist". | "Switch to the political map", "Show me the cultural view" |
| `set_style_preset`      | Switch the map's visual theme — same as the Options panel's Style Preset selector. One of: `default`, `ancient`, `gloom`, `pale`, `light`, `watercolor`, `clean`, `atlas`, `darkSeas`, `cyberpunk`, `night`, `monochrome` (case-insensitive). Delegates to `window.changeStyle(preset)` which applies the preset's style JSON across all SVG elements and re-renders burg icons / labels. Custom user-saved presets (`fmgStyle_*`) are out of scope. | "Switch to the ancient theme", "Use the monochrome style" |
| `get_map_info`          | Read a summary of the current map: name, seed, mapId, dimensions, year/era, and counts of states, provinces, burgs, religions, cultures, rivers, markers, zones, cells. | "What's this map called?", "How many states are there?", "Summarize the world" |
| `regenerate_map`        | Generate a new map (same as the "New Map" button, F2). Optional `seed` reproduces a specific map. Waits up to 60s for the `map:generated` event before returning. | "Generate a new map", "Regenerate with seed 12345" |
| `regenerate_emblems`    | Regenerate every coat of arms — same as the Tools panel's Regenerate Emblems button. Wipes all existing state / province / burg COA SVG elements, generates fresh `coa` objects for every active entity (with culture-appropriate heraldic kinship), and redraws the Emblems layer. Returns counts of how many active states / burgs / provinces were regenerated. | "Regenerate all emblems", "Reroll every coat of arms" |
| `regenerate_burg_coa`   | Re-roll the coat of arms for a single burg — same as the Regenerate button in the Burg Editor / Emblem Editor. Calls `COA.generate(parent, 0.3, 0.1, null)` using the burg's province (or state if no province) as heraldic parent, preserves the existing shield shape (or generates one via `COA.getShield`), writes `burg.coa`, and best-effort refreshes the `#burgCOA{i}` DOM node via `COArenderer.trigger`. Optional `shield` overrides the shape. Refuses removed or locked burgs. Use `regenerate_emblems` for the whole-map version. | "Reroll the emblem for Stormport", "Give burg 5 a new coat of arms with shield shape noldor" |
| `set_burg_coa_custom`   | Toggle the `burg.coa.custom` flag — the implicit "locked / hand-crafted" state the Emblem Editor sets when you upload a custom SVG / raster. Bulk regenerators (`regenerate_emblems`), single-burg regen (`regenerate_burg_coa` parent resolution), and `set_culture_shield` all skip any emblem where `coa.custom` is truthy. Pass `custom: true` to protect the emblem, `false` to allow regeneration. Requires an existing `burg.coa` (use `regenerate_burg_coa` first if missing). Refuses burg 0, removed burgs, and burgs locked via `set_entity_lock`. Idempotent. | "Lock the emblem on Stormport", "Protect burg 5's coat of arms from regeneration", "Unlock burg 12's emblem" |
| `set_state_coa_custom`  | Toggle the `state.coa.custom` flag — the implicit "locked / hand-crafted" state the Emblem Editor sets when you upload a custom SVG / raster for a state emblem. Bulk regenerators (`regenerate_emblems`) and the single-state regenerator (`regenerate_state_coa`) skip any emblem where `coa.custom` is truthy, and the renderer refuses to redraw it. Pass `custom: true` to protect the emblem, `false` to allow regeneration. Requires an existing `state.coa` (use `regenerate_state_coa` first if missing). Refuses state 0 (Neutrals), removed states, and states locked via `set_entity_lock`. Idempotent. Parallels `set_burg_coa_custom`. | "Lock the emblem on Altaria", "Protect state 3's coat of arms from regeneration", "Unlock state 7's emblem" |
| `set_province_coa_custom` | Toggle the `province.coa.custom` flag — the implicit "locked / hand-crafted" state the Emblem Editor sets when you upload a custom SVG / raster for a province. `regenerate_emblems` and `regenerate_province_coa` (parent resolution) both treat a `custom` parent / entry as opaque: the custom emblem is not mixed into child heraldry and is not overwritten by a re-roll that targets it. Pass `custom: true` to protect the emblem, `false` to allow regeneration. Requires an existing `province.coa` (use `regenerate_province_coa` first if missing). Refuses province 0, removed provinces, and provinces locked via `set_entity_lock`. Idempotent. Parallels `set_burg_coa_custom`. | "Lock the emblem on the Duchy of Rookwood", "Protect province 5's coat of arms from regeneration", "Unlock province 12's emblem" |
| `regenerate_state_coa`  | Re-roll the coat of arms for a single state — same as the Regenerate button in the Emblem Editor when a state is selected. States are top-level in the heraldry hierarchy, so `COA.generate(null, 0.3, 0.1, null)` is called with no parent coa. Preserves the existing shield shape (or generates one via `COA.getShield(state.culture, state.i)`), writes `state.coa`, and best-effort refreshes the `#stateCOA{i}` DOM node via `COArenderer.trigger`. Optional `shield` overrides the shape. Refuses state 0 (Neutrals), removed, and locked states. Use `regenerate_emblems` for the whole-map version and `regenerate_burg_coa` for the burg-level parallel. | "Reroll the emblem for Altaria", "Give state 3 a new coat of arms with shield shape noldor" |
| `regenerate_domain`     | Regenerate a single domain — same as one of the per-domain Regenerate buttons in the Tools panel. `domain` (case-insensitive) is one of: `rivers`, `routes`, `population`, `states`, `provinces`, `burgs`, `religions`, `cultures`, `military`, `ice`, `markers`. Delegates to the matching global (e.g. rivers → `regenerateRivers`, population → `recalculatePopulation`). Use `regenerate_map` for full re-gen and `regenerate_emblems` for coats of arms. | "Regenerate rivers", "Reroll the states", "Recalculate population" |
| `regenerate_zones`      | Regenerate zones — same as the Tools panel's Regenerate Zones button. Optional `multiplier` (0 – 100, default `1`) controls roughly how many zones are created (2 → ~2× the default, 0.5 → half). Delegates to `Zones.generate(multiplier)` and best-effort calls `drawZones()`. Returns the new active zone count. | "Regenerate zones", "Double the zones (multiplier 2)", "Halve the zones — multiplier 0.5" |
| `list_states`           | List states on the current map with name, fullName, form, type, color, culture, capital, burgs count, cells, area, and population. Paginated: `limit` 1–500 (default 100), `offset` ≥ 0. Skips the Neutrals placeholder. | "List the states", "Which state has the most burgs?" |
| `list_burgs`            | List burgs (cities/towns) with coords, population, state/culture names, capital/port flags, and type. Pagination (`limit` 1–500, `offset` ≥ 0) and filters: `state` (id or name), `capital_only`, `port_only`. | "List the biggest cities", "Show ports in Altaria", "Which burgs are capitals?" |
| `rename_burg`           | Rename a specific burg by id or case-insensitive current name. Updates the burg's label on the map automatically. | "Rename Stormport to Tidegarde", "Rename burg 7 to Arkhaven" |
| `regenerate_burg_name`  | Roll a fresh name for a burg — the "regenerate name" buttons in the Burg Editor. Default `mode=culture` uses `Names.getCulture(burg.culture)`; `mode=random` picks a random name-base via `Names.getBase`. Writes `burg.name` and best-effort updates the `#burgLabel{i}` SVG text. | "Reroll the name for Stormport", "Give burg 5 a random name (mode: random)" |
| `regenerate_all_burg_names` | Bulk-regenerate names for every non-locked, non-removed burg — the Burgs Overview's "Regenerate names" button. Default `mode=culture` uses `Names.getCulture(burg.culture)` per burg; `mode=random` picks a random name-base per burg. Preserves locked burgs (lock burgs first via `set_entity_lock`). Reports counts (`regenerated`, `skippedLocked`, `skippedRemoved`). | "Reroll every burg name", "Give all burgs random names (mode: random)" |
| `regenerate_all_state_names` | Bulk-regenerate short names for every non-locked, non-removed state — the States Editor's "Regenerate Names" button. Skips Neutrals (state 0). Default `mode=culture` calls `Names.getState(Names.getCultureShort(state.culture), state.culture)` per state; `mode=random` picks a random name-base per state. Writes `state.name` and calls `drawStateLabels()` once at the end to refresh every label. Lock states first via `set_entity_lock` to preserve them. Reports `renamed` / `skipped` lists. | "Reroll every state name", "Give all states random names (mode: random)" |
| `regenerate_state_name` | Roll a fresh short name for a state — the "regenerate name" buttons in the States Editor's name dialog. Default `mode=culture` calls `Names.getState(Names.getCultureShort(state.culture), state.culture)`; `mode=random` calls `Names.getState(Names.getBase(base), undefined, base)` with a random name-base. Writes `state.name` and best-effort calls `drawStateLabels([i])` to refresh. Rejects Neutrals (0). | "Reroll the name of the Kingdom of Altaria", "Give state 3 a random name (mode: random)" |
| `regenerate_province_name` | Roll a fresh short name for a province — the regenerate buttons in the province-name dialog. Default `mode=culture` uses the province's cell culture via `Names.getState(Names.getCultureShort(culture), culture)`; `mode=random` picks a random name-base. Writes both `province.name` AND `province.fullName` (recomposed as `"{name} {formName}"` matching the UI's `getFullName`). Best-effort updates the `#provinceLabel{i}` SVG. | "Reroll the name for the North Mark", "Give province 2 a random name" |
| `regenerate_all_province_names` | Bulk-regenerate short names for every non-locked, non-removed province (skips province 0) — parallels the Provinces Editor's bulk rename. Default `mode=culture` reads each province's cell culture (via `pack.cells.culture[center]`) and calls `Names.getState(Names.getCultureShort(culture), culture)` per province; `mode=random` picks a random name-base per province. Writes `province.name` AND `province.fullName` (recomposed as `"{name} {formName}"` / `"The {formName}"` matching the UI's `getFullName`). Best-effort updates every `#provinceLabel{i}` SVG. Lock provinces first via `set_entity_lock` to preserve them. Reports `renamed` / `skipped` lists. | "Reroll every province name", "Give all provinces random names (mode: random)" |
| `regenerate_all_culture_names` | Bulk-regenerate names for every non-locked, non-removed culture (skips Wildlands, culture 0). The Cultures Editor has a per-culture regenerate button (`cultureRegenerateName`) but no built-in bulk button; this AI tool applies the same algorithm across all active cultures. Default `mode=culture` calls `Names.getCultureShort(culture.i)` (which delegates to `Names.getBaseShort(culture.base)`); `mode=random` picks a random name-base per culture and calls `Names.getBaseShort(base)`. Writes `culture.name` (does NOT touch `culture.code` — parallels the per-culture editor button). Cultures have no on-map text labels, so no DOM refresh is needed; `drawCultures()` is called once at the end for parity. Cultures missing a namesbase are skipped with a "missing base" reason. Lock cultures first via `set_entity_lock` to preserve them. Reports `renamed` / `skipped` lists. | "Reroll every culture name", "Give all cultures random names (mode: random)" |
| `remove_burg`           | Delete a burg (same as Remove in the Burg Editor). Clears the cell link, note, emblem, and SVG icon/label. Refuses state capitals — run `set_state_capital` first to pick a new capital. | "Delete burg 7", "Remove Stormport" |
| `add_burg`              | Create a new burg at `(x, y)` — same as Tools panel's Add Burg. Validates land (cell height ≥ 20) and no existing burg at the target cell, then delegates to `Burgs.add([x, y])` which picks culture/state from the cell, generates a culture-appropriate name, populates, emblems, and wires into the routes network. Returns the new burg's id and key fields. Use `rename_burg` / `set_burg_culture` / `set_burg_type` etc. to customize after creation. | "Add a new burg at 500, 600", "Drop a burg at 100, 200" |
| `add_culture`           | Create a new culture centered on cell at `(x, y)` — same as Cultures Editor's Add Culture. Validates land and that the cell isn't already a culture center, then delegates to `Cultures.add(cellId)`. The new culture gets an auto-generated name, default expansionism 1, color, name-base, emblem shape, and Type 'Generic'. Follow up with `rename_culture` / `set_culture_type` / `set_culture_color` / `set_culture_base` / `set_culture_shield` to customize. | "Create a new culture at 400, 500", "Add a culture centered on cell near (100, 200)" |
| `add_religion`          | Create a new religion centered on cell at `(x, y)` — same as Religions Editor's Add Religion. Validates land and that the cell isn't already a religion center, then delegates to `Religions.add(cellId)`. Type / form / deity / expansion / color are auto-chosen from the cell's culture context (Folk if the culture has no Folk faith yet; otherwise Organized / Cult / Heresy). Follow up with `rename_religion` / `set_religion_type` / `set_religion_form` / `set_religion_deity` / `set_religion_expansion` / `set_religion_color`. | "Add a religion at 400, 500", "Create a new faith centered on cell (100, 200)" |
| `add_regiment`          | Create a new regiment for a state at `(x, y)` — same as the Regiment Editor's Add Unit mode. Snaps to the target cell's centroid, picks the naval flag from cell height (water → 1), assigns a fresh per-state id (last + 1), default icon "🛡️", zero units (`u: {}`, `a: 0`). Auto-names via `Military.getName`; legend generated via `Military.generateNote`. Works on Neutrals (state 0) too. Follow up with `set_regiment_unit` / `set_regiment_icon` / `rename_regiment`. | "Add a regiment for Altaria at 500, 600", "Create a new unit at 100, 200 for Neutrals" |
| `add_state`             | Create a new state by promoting an existing burg to be its capital — mirrors the States Editor's "Add state" button. Required: `capital` (burg id or name). Optional: `name` (auto via `Names.getState`), `color` (random via `getRandomColor`; default `#888888`), `type` (default `"Generic"`), `culture` (default = capital burg's culture), `form` (default `"Monarchy"`; `fullName` is recomposed as `"{form} of {name}"`). Mutates: pushes onto `pack.states`, sets `burg.capital = 1` + `burg.state = newI`, writes `pack.cells.state[burg.cell] = newI`. **Scope**: creates a **single-cell** state — NO territory expansion (no neighbours, no BFS). For a full territory re-grow, follow up with `regenerate_domain(domain="states")`. Best-effort calls `drawStates` / `drawStateLabels([newI])` / `drawBorders`. Refuses: removed burgs, burgs already a capital, burgs on water, unknown culture refs. | "Promote Stormport into its own state", "Make burg 5 the capital of a new kingdom", "Create a new state with capital Eldoria named Wyrmhold, color #aa8844, form Republic" |
| `add_province`          | Create a new province by promoting an existing burg to be its capital — mirrors the Provinces Editor's "Add province" button. Required: `capital` (burg id or name). Optional: `state` (defaults to the capital cell's state; if provided must match), `name` (defaults to the burg's name), `color` (defaults to a 20%-mix of the state color toward a random color via `d3.interpolate`, matching the editor; falls back to `getRandomColor` / `#888888`), `form` (default `"Province"`; `fullName` is composed as `"{name} {formName}"`). Mutates: pushes onto `pack.provinces`, pushes onto `pack.states[state].provinces`, writes `pack.cells.province[burg.cell] = newI`. Does **NOT** touch `burg.capital` (that flag is for state capitals only — province capitals are tracked via `province.burg`). **Scope**: creates a **single-cell** province — NO territory expansion (does not absorb neighbouring same-state cells like the editor's convenience loop does). Best-effort calls `drawProvinces` / `drawBorders`. Refuses: removed burgs, burgs on water, burgs in neutral lands (cellState === 0), cells already a province center, and state refs that don't match the capital cell's state. | "Make burg 7 the capital of a new Duchy", "Create a province in Altaria with capital Stormport", "Promote Eldoria into a new Barony" |
| `remove_province`       | Delete a province (same as the Provinces Editor trash icon). Zeroes every `pack.cells.province[cell]` that referenced it, splices it out of `pack.states[state].provinces`, writes `pack.provinces[i] = { i, removed: true }` (tombstone), removes the COA / province SVG elements, calls `unfog('focusProvince{i}')`, and calls `drawBorders()` to refresh. Rejects id 0 and already-removed provinces. | "Delete province 4", "Remove the North Mark" |
| `remove_religion`       | Delete a religion (same as the Religions Editor trash icon). Zeroes every `pack.cells.religion[cell]` that referenced it, writes `religion.removed = true` (tombstone — other fields preserved), and filters the removed id out of every other religion's `origins` array (resetting any emptied array to `[0]`). Best-effort removes the religion SVG elements. Response reports `cascadedOrigins`. Rejects "No religion" (id 0) and already-removed entries. | "Delete religion 4", "Remove the Old Faith" |
| `remove_culture`        | Delete a culture (same as the Cultures Editor trash icon). Reassigns every active burg and state with `culture === i` to 0 (Wildlands), zeroes matching `pack.cells.culture`, tombstones `pack.cultures[i].removed = true` (name preserved), and filters the removed id out of every other active culture's `origins` array (empty → `[0]`). Best-effort removes the `#culture{i}` / `#cultureCenter{i}` SVG elements. Response reports `cascadedOrigins`, `reassignedBurgs`, `reassignedStates`. Rejects Wildlands (id 0) and already-removed cultures. | "Delete culture 3", "Remove the Highlanders" |
| `remove_state`          | Delete a state (same as the States Editor trash icon). Big cascade: reassigns every burg in the state to neutral (clearing capital flag), zeroes `pack.cells.state`, tombstones every province of the state (including `cells.province` cleanup + province SVG), splices the state's regiments from the global notes, filters the state id out of every other state's `neighbors`, tombstones `pack.states[i]` (replaces the object with `{ i, removed: true }` — wipes the name, matches UI), best-effort removes the state / army SVG and calls `drawStates` / `drawBorders` / `drawProvinces`. Response reports all four cascade counts. Rejects Neutrals (id 0) and already-removed states. | "Delete state 3", "Remove the Kingdom of Altaria" |
| `merge_states`          | Absorb one or more states into a ruling state — same as the States Editor's Merge dialog. Ruling state keeps its name / color / capital. Cascades: the absorbed states' regiments are moved into the ruling state's `military` (with fresh per-state indices and matching notes renamed `regiment{old}-{oldI}` → `regiment{ruling}-{newI}`); burgs and provinces are reassigned; burg-capital flags on merged capitals are cleared; `pack.cells.state` entries are rewritten; each absorbed state is tombstoned. Best-effort: `unfog`, `States.getPoles`, `drawStates` / `drawBorders` / `drawProvinces` / `drawStateLabels`. Reports counts per cascade. Rejects Neutrals and `from`-lists that include the ruling state. | "Merge Brighton into Altaria", "Absorb states 2, 3, 5 into state 1" |
| `list_cultures`         | List cultures with name, color, type, cells, area, population, name base, shield, and code. Paginated (`limit` 1–500, `offset` ≥ 0). Skips the index-0 Wildlands placeholder. | "List the cultures", "Which culture has the biggest population?" |
| `rename_culture`        | Rename a culture by id or case-insensitive name. Regenerates the short code via the same abbreviation algorithm the Cultures Editor uses. Rejects the Wildlands placeholder. | "Rename Highlanders to Pinegarde", "Rename culture 2 to Wayfarers" |
| `list_religions`        | List religions with name, type (Folk/Organized/Cult/Heresy), form, deity, color, culture, cells, area, population, expansion, and code. Paginated (`limit` 1–500, `offset` ≥ 0). Skips the index-0 placeholder. | "List the religions", "Which faith has the most followers?" |
| `rename_religion`       | Rename a religion by id or case-insensitive name. Regenerates the short code via the same abbreviation algorithm the Religions Editor uses. Rejects the 'No religion' placeholder. | "Rename Old Faith to Wildshrine", "Rename religion 2 to Dawnkeepers" |
| `list_provinces`        | List provinces with name, fullName, formName, color, parent state, capital burg, and pole coordinates. Paginated (`limit` 1–500, `offset` ≥ 0) and optional `state` filter (id or name). Skips the index-0 placeholder. | "List provinces", "Which provinces are in Altaria?" |
| `list_markers`          | List map markers (points of interest) with type, icon, name/legend (from Notes), coords, cell, and pinned/lock flags. Paginated. Optional filters: `type` (case-insensitive exact) and `pinned_only`. | "List the markers", "Show me all the castles", "Which POIs are pinned?" |
| `set_marker_note`       | Rename a marker / POI and/or update its legend. Writes to the same global `notes[]` the Notes Editor uses; creates a note if one doesn't exist yet. Ref accepts marker id or current name (case-insensitive). `legend: ""` clears, whitespace-only is rejected. | "Rename the Rookhold marker to Dragon's Keep", "Add a legend to marker 5: 'Seat of the red king'" |
| `remove_marker`         | Delete a marker from the map (same as the trash icon in the Markers Overview). Removes the marker from `pack.markers`, its note from `window.notes`, and the SVG `#marker{i}` element. | "Delete marker 3", "Remove the Rookhold marker" |
| `set_marker_pinned`     | Pin or unpin a marker — same side-effect as the pin icon in the Markers Overview. Idempotent. Writes `marker.pinned`, keeps the `#markers` SVG group's `pinned` attribute in sync, and calls `drawMarkers()`. Matches by id or case-insensitive current note name. | "Pin the Rookhold marker", "Unpin marker 5" |
| `set_marker_lock`       | Lock or unlock a marker — same side-effect as the lock icon in the Markers Overview. Locked markers are preserved across regeneration. Idempotent. Writes `marker.lock` on lock, deletes the key on unlock. Matches by id or case-insensitive current note name. | "Lock the Rookhold marker", "Unlock marker 5" |

Metadata-only write: no redraw is triggered, matching the UI. Idempotent. Matches by id or case-insensitive current note name; rejects empty / whitespace-only types. | "Set marker 5's type to 'volcano'", "Change the Rookhold marker's type to lair" |
| `set_marker_icon`       | Change a marker's icon glyph or URL — the icon picker in the Markers Editor. Accepts any non-empty string (typically an emoji like `🌋` but URLs are allowed). Writes `marker.icon` and best-effort calls `drawMarkers()`. **Per-marker scope**: unlike the UI (which propagates to every same-type marker), this tool only updates the one marker — the AI can iterate if bulk behavior is wanted. | `set_marker_size`       | Change a marker's size — the Size input in the Markers Editor. Writes `marker.size` (default 30 if unset) and best-effort calls `drawMarkers()`. | `set_marker_icon_size`  | Change a marker's icon (glyph/emoji) size in pixels — the "Icon size" input in the Markers Editor. Distinct from `set_marker_size`, which controls the pin element size. Writes `marker.px` (default 12 if unset, allowed range `[2, 20]`) and best-effort calls `drawMarkers()`. **Per-marker scope** (same decision as `set_marker_icon`). Idempotent. | "Bump the Dragon Lair marker's icon to 16px", "Set marker 5's icon size to 8" |
| `set_marker_shift`      | Nudge a marker's icon inside the pin — the "Icon dx" / "Icon dy" number inputs in the Markers Editor (percentage offsets, default 50/50, allowed range `[0, 100]`). Distinct from `move_marker` (map position) and `set_marker_icon_size` (pixel size). At least one of `dx` / `dy` is required; the untouched field is preserved. Writes `marker.dx` / `marker.dy` and best-effort calls `drawMarkers()`. **Per-marker scope** (same decision as `set_marker_icon`). Idempotent. | "Nudge the Dragon Lair icon down (dy 70)", "Center marker 5's icon (dx 50, dy 50)" |
| `set_marker_pin`        | Change a marker's pin shape — the Pin Shape dropdown in the Markers Editor. ## Getting an API key

1. Sign in at <https://console.anthropic.com/>.
2. Go to **API Keys** and click **Create Key**.
3. Copy the generated key (it starts with `sk-ant-…`).

You will be billed for your own usage on your own Anthropic account.

## Using the chat in the app

1. Start the app (`npm run dev`, then open <http://localhost:5173>) or use a
   built version.
2. In the bottom-right corner, click the small circular **AI** button.
3. The first time the panel opens, an API-key row is expanded. Paste your
   `sk-ant-…` key and click **Save**. The key is stored in your browser's
   `localStorage` under the name `ai-chat-anthropic-api-key` — nothing is
   sent anywhere except directly to Anthropic.
4. Type a request in the input box (Shift+Enter for a newline, Enter to
   send). Example: `rename the map to Eldoria`.
5. You'll see:
   - your message (blue bubble),
   - any tool calls the AI makes (`→ set_map_name({"name":"Eldoria"})`,
     `✓ set_map_name → {"ok":true,...}`),
   - the assistant's reply.
6. Click the `–` in the header to collapse the panel back to the AI button.
7. Click the **⚙** in the header to show/hide the API-key row (to change or
   clear the stored key).

## Specifying the API key — other options

The built-in flow uses `localStorage`. For advanced usage:

- **Programmatic override**: call `setApiKey(key)` from the `src/ai/api-key`
  module (exported from `src/ai`).
- **Clearing the key**: call `clearApiKey()` from the same module, or open
  DevTools and run
  `localStorage.removeItem("ai-chat-anthropic-api-key")`.

There is intentionally no support for bundling an API key at build time —
that would ship the key to every visitor of the site.

## Security & limitations

- The API key is visible to any script running in the same page. Do not use
  the feature on a shared computer or on a page where you don't trust the
  other scripts.
- The key is sent directly from your browser to
  `https://api.anthropic.com/v1/messages`. The app has no backend
  intermediary.
- Anthropic requires the header
  `anthropic-dangerous-direct-browser-access: true` for browser calls. We
  set it automatically — it is a deliberate acknowledgement that direct
  browser calls are not the recommended deployment pattern.
- Each message consumes tokens on your Anthropic account. Keep conversations
  focused.

## Troubleshooting

- **"No API key set. Click ⚙…"** — you haven't saved a key yet.
- **401 / authentication_error** — the key is wrong, revoked, or your
  account has no credits.
- **429 / rate limit** — too many requests; slow down or upgrade your plan.
- **CORS errors in DevTools** — make sure you are not proxying the request
  through another host; the call must go directly to
  `api.anthropic.com`.
- **Tool call says `#mapName not found`** — the map input element is only
  present after the main map loads; reload the page and wait for the map to
  finish generating before issuing rename commands.

## Developing new tools

Use cases are implemented one per Ralph-loop iteration. See
`plan_N.md` and `tasks_N.md` at the root of the repo for the active plan.
To add a new tool:

1. Create a file in `src/ai/tools/your-tool.ts` that exports a `Tool`
   (matching the `Tool` interface in `src/ai/tools/index.ts`).
2. Register it from `src/ai/index.ts` inside `buildDefaultRegistry()`.
3. Add a unit test under `src/ai/tools/your-tool.test.ts`.
4. Update this README's tool table with the new capability.
