# Plan 147 â€” `regenerate_all_culture_names` AI tool

## Use case

Bulk-regenerate names for every non-locked, non-removed culture (skipping the index-0 "Wildlands" placeholder). Parallels the already-merged bulk name regenerators:
- `regenerate_all_state_names`
- `regenerate_all_burg_names`
- `regenerate_all_province_names`
- `regenerate_river_names`
- `regenerate_religion_names`

The Cultures Editor has a **per-culture** regenerate button (the `icon-cw` span wired to `cultureRegenerateName` in `public/modules/dynamic/editors/cultures-editor.js:347`) but **no bulk** "regenerate names" button â€” this tool is an AI-only convenience that calls the same underlying name-generator across every active culture.

## Name-generation algorithm (confirmed)

From `public/modules/dynamic/editors/cultures-editor.js:347` (`cultureRegenerateName`):

```js
function cultureRegenerateName() {
  const cultureId = +this.parentNode.dataset.id;
  const base = pack.cultures[cultureId].base;
  if (!nameBases[base]) return tip("Namesbase is not defined, please select a valid namesbase", false, "error", 5000);

  const name = Names.getCultureShort(cultureId);
  this.parentNode.querySelector("input.cultureName").value = name;
  pack.cultures[cultureId].name = name;
}
```

So the editor:
1. Reads `culture.base` to confirm a namesbase is defined.
2. Calls `Names.getCultureShort(cultureId)` â€” which in turn calls `Names.getBaseShort(pack.cultures[cultureId].base)` (see `src/modules/names-generator.ts:188`).
3. Writes `culture.name` (no COA / code / fullName side-effects).

At **initial generation** time, culture names come from hardcoded lists (per culture-set: "european", "oriental", "english", "antique", "highFantasy", "darkFantasy", "tribal") or from the "random" set which uses `Names.getBaseShort(rnd)` (`src/modules/cultures-generator.ts:784`). Either way, regenerating is just picking a fresh short name from the namesbase.

For this tool we expose **two modes** to match the parallel tools:
- `culture` (default) â€” `Names.getCultureShort(culture.i)` â€” uses the culture's own base.
- `random` â€” pick a random `nameBase` and call `Names.getBaseShort(baseIndex)`.

We match that behaviour here to preserve parity â€” renaming a culture via `rename_culture` regenerates the code because the single-culture AI tool reuses the editor's `abbreviate` flow; the bulk regen matches the editor's bulk button semantics (none exists, but the per-culture `cultureRegenerateName` doesn't touch `code`).

## Redraw

Culture **on-map labels** don't exist in this renderer â€” `drawCultures()` (`public/modules/ui/layers.js:434`) only fills region bodies by color. The Cultures Editor's legend / DOM row is regenerated when the editor re-opens; no live SVG text node depends on `culture.name`. So no DOM refresh is needed.

For parity with other bulk-name tools (states / provinces call `drawStateLabels`; rivers / religions call `drawRivers` / `drawReligions` as no-ops), we invoke `drawCultures?.()` once at the end as a best-effort no-op.

## Tool contract

Inputs:

```
{
  mode?: "culture" | "random"    // default "culture"
}
```

Outputs (matches other bulk-regenerate tools):

```
{
  ok: true,
  mode: "culture" | "random",
  renamed: [{ i, previousName, name }],
  skipped: [{ i, name, reason }]
}
```

Skip reasons:
- `"wildlands"` â€” culture 0 (the Wildlands sentinel).
- `"removed"` â€” tombstoned.
- `"locked"` â€” `culture.lock === true`.
- `"missing base"` â€” culture has no defined namesbase (the editor's guard).
- `"generate failed: <err>"` â€” `Names.getCultureShort` / `Names.getBaseShort` threw.
- `"empty generator output"` â€” generator returned empty / whitespace-only.
- `"apply failed: <err>"` â€” write threw.

## Runtime-seam split (pattern match for `regenerate-all-state-names`)

```ts
export const CULTURE_NAME_MODES = ["culture", "random"] as const;
export type CultureNameMode = (typeof CULTURE_NAME_MODES)[number];
export function resolveCultureNameMode(value: unknown): CultureNameMode | null;

export interface RegenerateAllCultureNamesCultureRef {
  i: number;
  name: string;
  base: number | null;
  lock?: boolean;
  removed?: boolean;
}

export interface RegenerateAllCultureNamesRuntime {
  list(): RegenerateAllCultureNamesCultureRef[];
  generate(mode: CultureNameMode, culture: RegenerateAllCultureNamesCultureRef): string;
  apply(i: number, name: string): void;
  redraw(): void;
}
```

Default runtime implementations:
- `list()` â€” reads `pack.cultures`, pulls `(i, name, base, lock, removed)`.
- `generate(mode, ref)` â€”
  - `culture` mode: calls `Names.getCultureShort(ref.i)`.
  - `random` mode: picks a random index into `nameBases`, calls `Names.getBaseShort(baseIndex)`.
- `apply(i, name)` â€” writes `pack.cultures[i].name = name` after re-validating the entry still exists and isn't removed.
- `redraw()` â€” `window.drawCultures?.()` best-effort (no-op for labels but matches parity).
